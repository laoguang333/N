# txt-reader

A self-hosted web-based TXT novel reader with Rust (Axum) backend and Vue 3 (Vite) frontend.

## Project Structure

```
frontend/          Vue 3.5 + Vite 7 SPA (JS, Composition API)
src/               Rust backend (Axum 0.8, SQLx 0.8, SQLite)
scripts/           Python 3 orchestration (dev.py, codex.py, check.py)
docs/              Architecture (architecture.md) and API (api.md) docs
config.toml        Runtime configuration (copy from config.example.toml)
```

## Build, Lint, Test Commands

### Frontend
```bash
npm install --prefix frontend
npm run test --prefix frontend          # Vitest unit tests (frontend/src/*.test.js)
npx playwright test --config frontend/playwright.config.js  # E2E tests (frontend/e2e/)
npm run build --prefix frontend         # Production build → frontend/dist/
```

### Backend
```bash
cargo build
cargo test
cargo clippy
```

### Full stack dev server
```bash
python scripts/codex.py
```

Always run `cargo clippy` and `npm run test --prefix frontend` after making changes. For frontend changes, also run `npx playwright test --config frontend/playwright.config.js` if UI flow is affected.

## Frontend Conventions (Vue 3 + Vite)

### Component Architecture
- **`<script setup>` exclusively** — never Options API, never `export default {}`.
- State uses `reactive()` for grouped objects, `ref()` for primitives/template refs.
- No Pinia or Vuex — all state is local reactive objects.
- No vue-router — routing via `window.location.hash` parsed in `App.vue`.
- Template events use kebab-case: `@click`, `@scroll.passive`.
- Components: PascalCase filenames (`AutoScroll.vue`).

### API Client (`frontend/src/api.js`)
- All HTTP calls through centralized `request(path, options)` wrapper around `fetch`.
- Naming: `get*()` for GET, `save*()` for PUT, `scan*()` for POST.
- Returns parsed JSON; throws `Error` with server message on failure.
- Progress saving: three strategies — regular PUT, keepalive POST, `sendBeacon`.
- API spec: read `docs/api.md` before adding/modifying endpoints.

### State Persistence
- `localStorage` keys prefixed `txt-reader-*` (kebab-case).
- Settings auto-persisted via deep watcher.
- Reading progress cached locally and synced to server.

### Virtual Scrolling
- `@tanstack/vue-virtual` for large book content.
- Recalculate layout when font/spacing settings change.
- Icons: `lucide-vue-next`.

### CSS
- Global styles in `frontend/src/styles.css`.
- Component-local styles with `<style scoped>`.
- Class naming: kebab-case, loosely BEM-inspired (`.reader-toolbar`, `.book-row`).

## Backend Conventions (Rust + Axum)

- Single binary — serves both JSON API and static frontend from `frontend/dist/`.
- SQLite with WAL mode, two tables: `books`, `reading_progress`.
- Endpoints in `src/api.rs`, models in `src/models.rs`.
- Error handling: `AppError` with `IntoResponse` (see `src/app_error.rs`).
- TXT file encoding: auto-detect with `chardetng`, convert Traditional→Simplified Chinese via `zhconv`.
- File dedup: SHA-256 hash for detecting moved/renamed files.

## Coding Style (All Languages)

### JavaScript
- `const`/`let`, never `var`.
- camelCase for functions/variables, UPPER_SNAKE for constants.
- Function names: verb-prefixed (`loadBooks`, `openBook`, `saveProgress`).
- API response keys use snake_case (backend convention).

### Rust
- Follow Clippy lints.
- Use `anyhow`/`thiserror` for error handling (already configured).
- `#[cfg(test)]` modules for inline unit tests.

### Python (scripts)
- Follow scripts' existing patterns (argparse, subprocess).
- Use `pathlib.Path` for file operations.

## Testing

- **Vitest**: Pure function unit tests, co-located (`reader.test.js` next to `reader.js`).
- **Playwright**: Browser-level E2E tests in `frontend/e2e/`.
- **Rust**: Inline `#[cfg(test)]` modules in `src/`.

When writing tests:
1. First read the existing test files to match conventions.
2. For E2E tests, mock backend API with `page.route()` — never depend on real backend.
3. For unit tests, test pure functions only (no component mounting).

## Key Constraints

- The backend renders frontend statically in production (files served from `frontend/dist/`).
- Proxy in dev: Vite dev server proxies `/api` to Rust backend.
- SSL certs in `certs/`, auto-generated by `scripts/codex.py`.
- Windows tray icon support: conditional compilation in `src/tray.rs`.
- PWA: `manifest.webmanifest` and service worker in `frontend/public/`.
